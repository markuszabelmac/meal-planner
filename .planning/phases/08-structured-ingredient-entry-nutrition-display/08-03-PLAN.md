**1. Create `src/lib/nutrition.ts`:**

Export a `computeNutritionTotals` function as described in the research. It takes an array of recipeIngredients (with nested ingredient) and a servings number. It:
- Iterates each row, skips if no `ingredient` linked
- Calls `unitToGrams(ri.amount, ri.unit)` — skips if null (stueck)
- Accumulates kcal, protein, fat, carbs weighted by `grams / 100`
- Tracks `hasAnyData` boolean and `hasSkippedStueck` boolean
- If `!hasAnyData`, returns `null` (signal: no nutrition to show)
- Divides totals by `servings` (default 1 if null/0)
- Rounds kcal to integer, macros to 1 decimal place
- Returns `{ kcal, protein, fat, carbs, hasSkippedStueck }` or `null`

Import `unitToGrams` and `Unit` from `@/lib/units`. Define an input type for the function parameter matching the Prisma include shape.

**2. Extend recipe detail page (`src/app/(app)/rezepte/[id]/page.tsx`):**

- Update the `prisma.recipe.findUnique` include to add `recipeIngredients: { include: { ingredient: { select: { id: true, name: true, kcalPer100g: true, proteinPer100g: true, fatPer100g: true, carbsPer100g: true } } }, orderBy: { createdAt: "asc" } }`
- Import `computeNutritionTotals` from `@/lib/nutrition`
- After fetching recipe, compute: `const nutrition = computeNutritionTotals(recipe.recipeIngredients, recipe.servings)`
- Render the nutrition table AFTER the meta badges section and BEFORE the ingredients section. Only render when `nutrition !== null`.
- Use the table markup from the research: heading "Nahrwerte pro Portion", a bordered table with 4 rows (Kalorien, Protein, Fett, Kohlenhydrate), right-aligned values with units.
- If `nutrition.hasSkippedStueck`, show a small footnote: `* Zutaten mit Einheit "Stuck" sind nicht eingerechnet.`
- For the ingredients section: show structured ingredient list (name + amount + unit) when `recipe.recipeIngredients.length > 0`, otherwise show the freetext fallback as it is now. This means: when structured ingredients exist, render them as a clean list (e.g., "200 g Hackfleisch, 500 ml Milch") instead of the freetext. When no structured ingredients, show freetext as before.

**3. Extend recipe detail modal (`src/components/recipe-detail-modal.tsx`):**

- Extend the `Recipe` type to include `recipeIngredients` (optional array matching the API response shape)
- Import `computeNutritionTotals` from `@/lib/nutrition`
- After recipe loads, compute nutrition from `recipe.recipeIngredients` (if present)
- Render the same nutrition table inline (same HTML structure/classes), placed after meta badges and before ingredients
- The API already returns recipeIngredients (from Plan 01 GET changes), so no additional fetch needed
- For ingredients display: same logic as detail page — show structured list when available, freetext fallback otherwise
- Import `Unit` type for the nutrition function — since this is a client component, `computeNutritionTotals` must work client-side. The function only uses `unitToGrams` which is a pure function with no server dependencies, so this works fine.

**Styling:** Follow existing project patterns — `rounded-lg border border-border bg-card`, `text-sm`, `px-4 py-2` table cells. Heading uses `font-semibold`. Footnote uses `text-xs text-muted`.
