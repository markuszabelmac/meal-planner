**2. Define row state type and unit labels:**
```typescript
type IngredientRowState = {
  id: string;                    // crypto.randomUUID() — stable React key
  searchText: string;            // controlled input for autocomplete
  ingredientId: string | null;   // set on selection
  ingredientName: string;        // display name after selection
  amount: string;                // string for controlled input
  unit: string;                  // Unit enum value, default "g"
  suggestions: { id: string; name: string }[];
  showDropdown: boolean;
};

const UNIT_LABELS: Record<string, string> = {
  g: "g", kg: "kg", ml: "ml", l: "l",
  stueck: "Stuck", el: "EL", tl: "TL", prise: "Prise",
};
```

**3. Add state initialization in RecipeForm:**
```typescript
const [ingredientRows, setIngredientRows] = useState<IngredientRowState[]>(() => {
  if (initial?.recipeIngredients?.length) {
    return initial.recipeIngredients.map(ri => ({
      id: crypto.randomUUID(),
      searchText: ri.ingredientName,
      ingredientId: ri.ingredientId,
      ingredientName: ri.ingredientName,
      amount: ri.amount,
      unit: ri.unit,
      suggestions: [],
      showDropdown: false,
    }));
  }
  return [];
});
```

**4. Autocomplete fetch function:**
- On input change for `searchText`, if text.length >= 2, fetch `/api/ingredients?search=<text>`
- Track a `requestId` counter (useRef) per row to avoid stale results overwriting newer ones
- On fetch return, set `suggestions` array on that row and `showDropdown: true`
- On `searchText` change, clear `ingredientId` (deselect) — user must re-pick

**5. Suggestion selection:**
- On `onMouseDown` (NOT onClick — prevents blur from hiding dropdown before click registers) of a suggestion item:
  - Set `ingredientId`, `ingredientName`, `searchText` to the selected ingredient
  - Set `showDropdown: false`, clear `suggestions`

**6. Dropdown close behavior:**
- On input `onBlur`: set `showDropdown: false` with a 150ms setTimeout to let mousedown fire first
- On Escape key in input: set `showDropdown: false`

**7. Row add/remove:**
- "Zutat hinzufugen" button adds a new empty row with `crypto.randomUUID()` id, default unit "g"
- Each row has an X button to remove it from the array
- Use the row's `id` as React key (never array index)

**8. Render the editor section:**
Place the structured ingredient editor section BEFORE the existing freetext "Zutaten" textarea. Add a heading "Strukturierte Zutaten" with a brief label. Each row renders:
- Text input for ingredient search (with dropdown suggestions positioned absolute below)
- Number-like input for amount (type="text" with inputMode="decimal" for mobile — NOT type="number" to avoid scroll-to-change behavior)
- Select dropdown for unit (from UNIT_LABELS)
- X remove button

Style with Tailwind matching project conventions: `rounded-lg border border-border`, same spacing as other form sections. Rows stack vertically. Each row is a flex row with gap-2.

The suggestions dropdown: `absolute z-10 mt-1 max-h-48 overflow-y-auto rounded-lg border border-border bg-card shadow-lg`. Each suggestion item: `px-3 py-2 text-sm cursor-pointer hover:bg-background`.

**9. Extend handleSubmit:**
- Build `recipeIngredients` array from `ingredientRows`: filter to rows where `ingredientId` is truthy AND `parseFloat(amount)` is a valid positive number
- Map to `{ ingredientId, amount: parseFloat(row.amount), unit: row.unit }`
- Include `recipeIngredients` in the JSON body sent to the API

**10. Keep freetext field:**
The existing "Zutaten" textarea stays exactly as-is below the structured editor. Rename its label from "Zutaten" to "Zutaten (Freitext)" to distinguish. No other changes to the freetext field.

**Key pitfall avoidance:**
- Use `onMouseDown` on suggestion items, not `onClick` (blur fires before click)
- Use stable UUID keys, never array index
- Amount as string in state, only parseFloat on submit
- Filter invalid rows on submit (empty amount, no ingredientId)
