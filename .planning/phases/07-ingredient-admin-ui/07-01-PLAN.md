  <action>
Create two API route files mirroring the recipe API pattern exactly (auth check via `const session = await auth()`, NextResponse.json responses, same error format).

**src/app/api/ingredients/route.ts — GET and POST:**

GET handler:
- Parse `search` query param from URL
- If `search.length >= 2`: use `prisma.$queryRaw` with `Prisma.sql` tagged template for pg_trgm fuzzy search. Query must SELECT id, name, name_en AS "nameEn", kcal_per_100g AS "kcalPer100g", protein_per_100g AS "proteinPer100g", fat_per_100g AS "fatPer100g", sat_fat_per_100g AS "satFatPer100g", carbs_per_100g AS "carbsPer100g", sugar_per_100g AS "sugarPer100g", fiber_per_100g AS "fiberPer100g", is_custom AS "isCustom" FROM ingredients. Use `WHERE name % ${search} OR name_en % ${search} OR name ILIKE ${'%' + search + '%'}` to combine fuzzy + substring match. ORDER BY `GREATEST(similarity(name, ${search}), similarity(COALESCE(name_en, ''), ${search})) DESC`. LIMIT 50.
- If no search or search.length < 2: use `prisma.ingredient.findMany` with `orderBy: [{ isCustom: "desc" }, { name: "asc" }]`, `take: 100`, selecting all nutrition fields plus id, name, nameEn, isCustom.
- Return 401 if no session.

POST handler:
- Parse JSON body: { name, nameEn, kcalPer100g, proteinPer100g, fatPer100g, satFatPer100g, carbsPer100g, sugarPer100g, fiberPer100g }
- Validate: name is non-empty string (trim it), kcalPer100g is a number >= 0
- Create with `prisma.ingredient.create`, always set `isCustom: true`
- Return 201 with created ingredient
- Return 401 if no session, 400 on validation failure

**src/app/api/ingredients/[id]/route.ts — PUT and DELETE:**

Both handlers: auth check, `const { id } = await params` (Next.js 16 async params pattern), find ingredient first with `findUnique`.

PUT handler:
- Accept same body fields as POST (name, nameEn, all nutrition fields)
- Validate name non-empty, kcalPer100g >= 0
- Update with `prisma.ingredient.update`
- Return updated ingredient
- Return 404 if not found

DELETE handler:
- Check `ingredient.isCustom` — if false, return 403 with error "USDA-Zutaten koennen nicht geloescht werden"
- If custom, check if ingredient has linked RecipeIngredients: `prisma.recipeIngredient.count({ where: { ingredientId: id } })`. If count > 0, include a note in the response that linked recipes will lose the structured ingredient link (the schema uses onDelete: SetNull so deletion is safe, but inform the caller).
- Delete with `prisma.ingredient.delete`
- Return `{ success: true, detachedRecipes: count }`
- Return 404 if not found

Import pattern: `import { auth } from "@/lib/auth"`, `import { prisma } from "@/lib/db"`, `import { Prisma } from "@/generated/prisma"`. Use `import type { NextRequest } from "next/server"` only if needed; the recipe routes use plain `Request` type.

Route handler signatures must match Next.js 16 pattern: `export async function DELETE(_req: Request, { params }: { params: Promise<{ id: string }> })`.
