<task type="auto">
  <name>Task 1: Update Prisma schema with new models, enums, and MealPlan migration</name>
  <files>prisma/schema.prisma, prisma.config.ts</files>
  <action>
1. Update `prisma/schema.prisma`:

   Add `MealType` enum with values: `fruehstueck`, `mittag`, `abend`, `snacks`. Map to `@@map("MealType")`.

   Add `Unit` enum with values: `g`, `kg`, `ml`, `l`, `stueck`, `el`, `tl`, `prise`. Map to `@@map("Unit")`.

   Add `Ingredient` model (mapped to `"ingredients"`):
   - `id String @id @default(cuid())`
   - `name String` (German name, primary)
   - `nameEn String? @map("name_en")` (English USDA name)
   - `fdcId Int? @unique @map("fdc_id")` (USDA FDC ID for idempotent seed)
   - `kcalPer100g Float @map("kcal_per_100g")`
   - `proteinPer100g Float? @map("protein_per_100g")`
   - `fatPer100g Float? @map("fat_per_100g")`
   - `satFatPer100g Float? @map("sat_fat_per_100g")`
   - `carbsPer100g Float? @map("carbs_per_100g")`
   - `sugarPer100g Float? @map("sugar_per_100g")`
   - `fiberPer100g Float? @map("fiber_per_100g")`
   - `isCustom Boolean @default(false) @map("is_custom")`
   - `createdAt DateTime @default(now()) @map("created_at")`
   - `updatedAt DateTime @updatedAt @map("updated_at")`
   - `recipeIngredients RecipeIngredient[]`

   Add `RecipeIngredient` model (mapped to `"recipe_ingredients"`):
   - `id String @id @default(cuid())`
   - `recipeId String @map("recipe_id")`
   - `ingredientId String? @map("ingredient_id")` (nullable for AI-estimated fallback in Phase 9)
   - `amount Float`
   - `unit Unit`
   - `estimatedKcal Float? @map("estimated_kcal")`
   - `estimatedProtein Float? @map("estimated_protein")`
   - `estimatedFat Float? @map("estimated_fat")`
   - `estimatedCarbs Float? @map("estimated_carbs")`
   - `createdAt DateTime @default(now()) @map("created_at")`
   - Relations: `recipe Recipe @relation(fields: [recipeId], references: [id], onDelete: Cascade)`, `ingredient Ingredient? @relation(fields: [ingredientId], references: [id], onDelete: SetNull)`

   Add `recipeIngredients RecipeIngredient[]` relation to the existing `Recipe` model.

   Update `MealPlan` model:
   - Add `mealType MealType @map("meal_type")`
   - Change `@@unique([date, forUserId])` to `@@unique([date, forUserId, mealType])`

2. Run `npx prisma migrate dev --create-only --name add_nutrition_schema` to generate migration SQL without applying.

3. Edit the generated migration SQL file to handle the MealType enum correctly:
   - The CREATE TYPE statements for MealType and Unit will be generated outside BEGIN by Prisma.
   - Add `meal_type` column as NULLABLE first (no DEFAULT).
   - Add `UPDATE "meal_plans" SET "meal_type" = 'abend' WHERE "meal_type" IS NULL;` after the column is added.
   - Add `ALTER TABLE "meal_plans" ALTER COLUMN "meal_type" SET NOT NULL;` after the UPDATE.
   - Ensure the old unique constraint `meal_plans_date_for_user_id_key` is dropped and the new `meal_plans_date_for_user_id_meal_type_key` is added.
   - Add `CREATE EXTENSION IF NOT EXISTS pg_trgm;` at the top of the migration.
   - Add GIN indexes after the Ingredient table creation:
     `CREATE INDEX ingredients_name_trgm_idx ON "ingredients" USING GIN ("name" gin_trgm_ops);`
     `CREATE INDEX ingredients_name_en_trgm_idx ON "ingredients" USING GIN ("name_en" gin_trgm_ops);`

4. Apply the migration: `npx prisma migrate dev`

5. Update `prisma.config.ts`: Add `seed` to the `migrations` object:
   `seed: "node --experimental-transform-types prisma/seed.ts"`
