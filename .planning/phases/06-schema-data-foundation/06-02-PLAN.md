<task type="auto">
  <name>Task 2: Create and run seed script with AI translation</name>
  <files>prisma/seed.ts</files>
  <action>
Create `prisma/seed.ts` that:

1. **Imports:** Use relative path `../src/generated/prisma/client` for PrismaClient (NOT @/ alias). Use `@prisma/adapter-pg` for PrismaPg adapter. Import OpenAI from `openai` package (already installed).

2. **PrismaClient setup:** Create a new PrismaClient instance with PrismaPg adapter using `process.env.DATABASE_URL`.

3. **Load USDA data:** Read the JSON file from `prisma/seed-data/`. Parse the top-level array (key name determined in Task 1). Handle the ~600MB memory usage — `readFileSync` + `JSON.parse` is acceptable for a one-time seed.

4. **Filter by category:** Use the ALLOWED_CATEGORIES set from the research (14 categories: Vegetables, Fruits, Beef, Poultry, Pork, Lamb/Veal/Game, Fish/Seafood, Dairy/Egg, Legumes, Cereal/Pasta, Nuts/Seeds, Fats/Oils, Spices/Herbs, Baked Products). Filter foods where `foodCategory.description` is in the set.

5. **Extract nutrients:** For each food, extract nutrient values using nutrient IDs:
   - energy (kcal): 1008
   - protein: 1003
   - fat: 1004
   - saturated fat: 1258
   - carbs: 1005
   - sugars: 2000
   - fiber: 1079
   Skip foods where kcal is missing (null/undefined).

6. **AI batch translation:** Batch 50 English food names per OpenAI API call. Use `gpt-4o-mini` model. System prompt: "Translate the following food ingredient names from English to German. Return a JSON array of strings in the same order. Use common German food names (e.g., 'Chicken breast, raw' -> 'Hahnchenbrust, roh'). If no standard German translation exists, keep the English term." Use `response_format: { type: "json_object" }`. Parse the response and map translations back to foods.

7. **Upsert:** For each translated food, run `prisma.ingredient.upsert` with `where: { fdcId }`. Set `isCustom: false`. Map all nutrient fields. Use `createMany` or batch upserts for performance if possible, otherwise sequential upserts are fine for a one-time script.

8. **Logging:** Print progress every 100 items. Print total count at end. Print any skipped items (missing kcal, translation failure).

9. **Error handling:** Wrap in try/catch. On translation API failure for a batch, retry once, then skip that batch with a warning. Always `prisma.$disconnect()` in finally block.

10. **Idempotency:** The `fdcId` unique constraint ensures re-running updates existing records rather than creating duplicates.

IMPORTANT: The seed script runs outside Next.js context. Use `import "dotenv/config"` at the top for env vars. Use relative paths for all local imports.
  </action>
